Em Git, um fork é uma cópia de um repositório que é criada em uma conta Git diferente . Isso permite que você trabalhe em um projeto sem interferir no repositório original e sem precisar solicitar acesso ao proprietário do repositório. Você pode usar um fork para fazer alterações no código e, em seguida, enviar uma solicitação pull para o repositório original, comercialmente denominado "upstream", para que o proprietário do repositório possa revisar e incorporar suas alterações. Forking é frequentemente usado em projetos de código aberto, onde muitas pessoas podem querer contribuir para o projeto.
